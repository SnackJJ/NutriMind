### Task 6.2: Agentic 能力边界分析 (1.5 天)

回答 PRD 核心研究问题: *RL 能将 3B 模型的 agentic 能力推到多远？安全边界在哪里？*

- [ ] **6.2.1** 设计边界探测测试集 (50+ 条)
  - 逐步增加约束复杂度: 2 约束 → 3 约束 → 4+ 约束
  - 逐步增加规划跨度: 单餐 → 一天 → 一周 → 一月

- [ ] **6.2.2** 运行边界测试，记录每个场景的:
  - escalation decision (是否升级)
  - 未升级时的回答质量 (LLM-judge)
  - 工具使用合理性
  - 推理链质量 (`<think>` 内容分析)

- [ ] **6.2.3** 绘制能力曲线
  - X 轴: 任务复杂度 (约束数/规划跨度)
  - Y 轴: 成功率 / 回答质量
  - 找到 "性能断崖" — 能力急剧下降的临界点

- [ ] **6.2.4** 分析边界特征
  - 模型正确触发 T4 安全声明的场景特征 → 学到了哪些关键信号
  - 模型错误触发 T4（过度保守， Type A 场景误触安全声明）
  - 模型未能触发 T4（漏报高危场景 → 最重要的失败模式）
  - 模型独立完成的最复杂场景 → 能力上限

- [ ] **6.2.5** 输出边界分析报告
  - 3B 模型可靠处理的复杂度上限
  - RL 对边界的推移效果 (SFT 断崖点 vs SFT+GRPO 断崖点)
  - Escalation 判断模式分析
